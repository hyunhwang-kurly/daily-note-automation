# 데일리 작업 로그 기록

현재 세션에서 **실제로 한 작업**을 요약해 오늘 데일리 노트에 남깁니다.

## 절차

1. 이 대화에서 한 작업을 돌아보고 **3~7개의 간결한 한국어 불릿**으로 요약한다. 우선순위:
   - 무엇을 했는가 — 기능/수정/리팩터 등 **결과 중심**, "~함" 체
   - 관련 프로젝트·주요 파일
   - 커밋/PR/배포가 있었다면 해시·번호
   - 중요한 결정이나 남은 할 일
   원시 명령 나열·잡담은 제외하고, **나중에 봐도 의미 있는 수준**으로.

2. 아래 명령으로 기록한다. **불릿 하나당 인자 하나**로 전달:
   ```bash
   node /Users/g02t3nc7cd/Work/daily-note-automation/bin/daily-note.js log "불릿1" "불릿2" "불릿3"
   ```
   - 큰따옴표 안 내용은 셸에 안전하게(따옴표/특수문자 escape) 넣는다.
   - 줄이 많거나 특수문자가 많으면 stdin 사용:
     ```bash
     printf '%s\n' "불릿1" "불릿2" | node /Users/g02t3nc7cd/Work/daily-note-automation/bin/daily-note.js log
     ```

3. 실행 결과(`작업 로그 N건 기록 → 경로`)를 사용자에게 한 줄로 보고한다.

## 동작/주의

- 오늘 날짜 노트의 **Work 칸 하위 `🤖 Claude Code` 그룹**에 타임스탬프(HH:MM)와 함께 추가된다. 파일/오늘 섹션이 없으면 자동 생성.
- **메일은 보내지 않는다** — 노트 파일만 갱신.
- 요약은 **사실 기반**. 하지 않은 일을 지어내지 않는다.
- 경로(`/Users/g02t3nc7cd/Work/daily-note-automation`)가 다르면 실제 설치 경로로 바꿔 실행한다.
